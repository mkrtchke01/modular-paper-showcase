# Сайт-пример — разбор

**URL:** https://readymag.website/u2184403773/5612656/
**Платформа:** Readymag (рендерится через JS). Тема примера: **фестиваль «Маски мира»** — лендинг с продажей билетов.

> Заказчику нравится **структура и подача** этого сайта. Нужно повторить структуру/композицию, но: **наши цвета**, **наши элементы конструктора вместо треугольников**, **без анимации вращения**, **обычный курсор**.

Скриншоты секций лежат рядом:
- `example-1-hero.jpg` — первый экран
- `example-2-tickets.jpg` — карточки (аналог карточек товаров)
- `example-3-order.jpg` — форма заказа

Сочетание: **насыщенный синий фон + крупная светлая типографика + жёлтые и розовые геометрические акценты**.

## Типографика

- Шрифт: **Arimo** (sans-serif, метрически совместим с Arial/Helvetica). Запасной — Arial, sans-serif.
- Заголовки: **очень крупные, заглавными, жирные** (hero ~73–95px), плотный leading.
- Подзаголовки/навигация: заглавные, разрядка, средний вес.
- Body: 15–21px, обычный вес.

## Геометрия / декор (ключевая особенность)

- Весь визуал построен на **треугольниках** (танграм-стиль): мозаичные колонны из сине-жёлто-розово-белых треугольников по краям экранов, треугольные «буллеты» в списках, уголки на карточках, фоновый паттерн из треугольников, лицо «маски» собрано из розовых треугольников.
- **→ В нашем проекте треугольники заменяем на элементы конструктора заказчика** (модульные «бумажные» фигуры).
- В оригинале фигуры **вращаются/анимируются** (`rotate`, scroll/hover/load анимации) — **по ТЗ это убираем**.

## Структура (3 секции / scroll-страницы)

### Секция 1 — Hero + «О фестивале» + программа (длинный скролл)
- **Навигация** (верх): `О ФЕСТИВАЛЕ` · `ПРОГРАММА` · `БИЛЕТЫ` · `КОНТАКТЫ` + бургер-иконка справа.
- **Первый экран:** слева вертикальная мозаика из треугольников; справа огромный заголовок **«МАСКИ МИРА»**, подзаголовок **«ФЕСТИВАЛЬ КУЛЬТУР»**, жёлтая кнопка **«КУПИТЬ БИЛЕТ»**.
- Блок места/дат: «Место проведения… Даты: 7–9 июля 2025».
- Блок **«ЧТО ТАКОЕ ФЕСТИВАЛЬ "МАСКИ МИРА"?»** — описательный текст (аналог раздела **«О нас»** для заказчика).
- Блок **«СОБЕРИ СВОЮ МАСКУ»**.
- **Программа** — длинный список пунктов: тире + название события, площадка, описание (Открытие фестиваля, Парад масок, Концерт, Маскарад для детей, Мастер-классы, и т.д.).

### Секция 2 — Билеты (= аналог карточек товаров)
- Заголовок-якорь `Билеты`, жёлтый треугольный маркер.
- **Три карточки в ряд:** «Стандарт», «Все дни», «VIP».
  - Карточка: светлый фон `#EEF0EF`, **жёлтый треугольный уголок** снизу, заголовок синим с подчёркиванием, список преимуществ с **треугольными буллетами**, цена внизу крупно (`1000 ₽`, `2900 ₽`, `3500 ₽`).
- Справа — крупное **«лицо-маска»** из розовых треугольников на жёлтом фоне.
- **→ Это прямой образец «карточек товаров». По ТЗ при наведении карточка должна слегка увеличиваться (`scale`).**

### Секция 3 — Форма заказа
- Фон — **паттерн из сине-белых треугольников** на всю ширину.
- По центру **жёлтая карточка-форма «Ваш заказ»**: поля `Имя и фамилия`, `Email`, `Номер телефона`, строка `Всего: 3500 ₽`, синяя кнопка **«Перейти к оплате»**.
- (У заказчика это место займёт раздел **«Контакты»**.)

## Что меняем относительно примера (по ТЗ)
1. Цвета → фирменные цвета заказчика (синюю/жёлтую/розовую палитру заменить).
2. Треугольники → элементы конструктора заказчика.
3. Убрать анимацию вращения фигур.
4. Курсор — стандартный (в примере он, возможно, кастомный).
5. Контент (тексты, товары, контакты) — от заказчика.
6. Hover-карточек — добавить лёгкий `scale` (в примере не было).
