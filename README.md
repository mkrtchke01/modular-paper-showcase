# Modular Paper Showcase

Одностраничный адаптивный лендинг-каталог на чистых HTML, CSS и JavaScript. Визуальный язык построен на крупных editorial-заголовках и модульных «бумажных» деталях конструктора.

## Структура

```text
index.html        — контент и семантическая разметка
css/styles.css    — дизайн, адаптивность и состояния
js/main.js        — мобильное меню, reveal-эффекты и текущий год
assets/           — папка для будущих изображений товаров
```

## Локальный запуск

Сайт не требует сборки. Откройте `index.html` в браузере или запустите локальный сервер:

```powershell
python -m http.server 8080
```

После этого откройте `http://localhost:8080`.

## Как заменить контент

- **Цвета:** измените переменные в начале `css/styles.css`. Блок помечен комментарием `ПАЛИТРА`.
- **Товары:** отредактируйте карточки внутри `<div class="catalog-grid">` в `index.html`.
- **Изображения:** замените декоративный блок `.product-visual` на `<img>` с заполненным `alt` или добавьте изображение внутрь него.
- **Контакты:** замените placeholders в секции `<section id="contacts">`.
- **Название и тексты:** весь видимый контент находится в `index.html`.

## Публикация на GitHub Pages

1. Создайте публичный репозиторий на GitHub и загрузите в него файлы проекта.
2. Откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите **Deploy from a branch**.
4. Выберите ветку `main`, папку `/ (root)` и нажмите **Save**.
5. Через несколько минут сайт будет доступен по адресу `https://USERNAME.github.io/REPOSITORY/`.

GitHub Pages бесплатно размещает статический сайт без ограничения в один месяц. Альтернативы: Netlify Free и Vercel Hobby.

## Доступность и производительность

В проекте предусмотрены семантическая разметка, keyboard focus, skip-link, адаптивная навигация, отсутствие горизонтального скролла и поддержка `prefers-reduced-motion`. Внешние библиотеки и тяжелые изображения не используются.
