# Бумагу, пожалуйста — лендинг

Одностраничный адаптивный сайт студии упаковки подарков и бумажного творчества.
Чистые HTML, CSS и JavaScript, без сборки. Тёмная тема, логотип по центру первого
экрана, фирменные «бумажные» элементы конструктора вокруг, карточки товаров.

> Полный контекст задачи и материалов — в [`PROJECT_BRIEF.md`](PROJECT_BRIEF.md).

## Структура

```text
index.html        — разметка (Hero, О нас, Каталог, Контакты)
css/styles.css    — стили, адаптив, состояния (палитра — в начале файла)
js/main.js        — мобильное меню, фон шапки при скролле, reveal, год
assets/           — оптимизированные логотип и фигурки
  logo.png        — логотип «Бумагу, пожалуйста» (из «для лого-02»)
  fig-*.png       — плоские элементы конструктора (синий/зелёный/жёлтый/малиновый)
  render/*.png    — 3D-рендеры для карточек и блока «О нас» (уже сжаты для веба)
```

Исходные тяжёлые материалы заказчика лежат в `Исходники от заказчика/`
(`Logos/`, `Renders/`) — в продакшен не идут, используются только как источник ассетов.

## Локальный запуск

Сборка не нужна. Откройте `index.html` в браузере или запустите сервер:

```powershell
python -m http.server 8080
```

Затем откройте `http://localhost:8080`.

## Что ещё заменить (плейсхолдеры)

- **«О нас»** — текст написан как временный (помечен `TODO` в `index.html`), заменить на финальный от заказчика.
- **Товары** — 6 карточек с примерным наполнением (помечены `TODO`); заменить названия, описания, цены и изображения.
- Контакты и фирменные цвета/логотип/фигурки — уже реальные.

## Соответствие ТЗ

- Логотип по центру на чёрном фоне, элементы конструктора расставлены по краям **сбалансированно** (равный визуальный вес слева/справа).
- Вместо треугольников из сайта-примера — фирменные элементы конструктора заказчика.
- **Нет анимации вращения** (только мягкое появление и едва заметное «дыхание»).
- Курсор **обычный**, стандартный.
- Карточки товаров при наведении **немного увеличиваются** (`scale`).
- Разделы: «О нас», «Каталог/Товары», «Контакты».

## Публикация на бесплатном хостинге

Подходит любой статический хостинг без срока «засыпания»:

- **GitHub Pages:** залить файлы в публичный репозиторий → Settings → Pages → Deploy from a branch → `main` / `/ (root)` → Save. Адрес вида `https://USERNAME.github.io/REPO/`.
- **Netlify Free / Cloudflare Pages:** перетащить папку проекта или подключить репозиторий.

Все три бесплатно держат статический сайт без ограничения в один месяц.

## Доступность и производительность

Семантическая разметка, skip-link, keyboard focus, адаптивная навигация,
отсутствие горизонтального скролла, поддержка `prefers-reduced-motion`,
`loading="lazy"` на изображениях, оптимизированные ассеты, без внешних библиотек.
